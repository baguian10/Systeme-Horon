1. Rôle et responsabilités du Super Administrateur

Le Super Administrateur (Niveau 0) est le garant du fonctionnement technique et de gouvernance de la plateforme. Il dispose d'un accès total. Il met en œuvre les décisions des juges sans les prendre.

DomaineResponsabilité
ComptesCréer/gérer tous les comptes (juges, agents, administrateurs délégués).
MatérielParc de bracelets, balises BLE, SIM, assignation, maintenance.
PérimètresTracer et valider techniquement les obligations de zone définies par les juges.
SystèmeParamètres, rétention, sécurité, audit, sauvegarde.
SupervisionSanté de la flotte, connectivité, incidents techniques.
Principe. Le juge décide (mesure, zones, sanction) ; le Super Administrateur exécute et maintient. Cette séparation est inscrite dans les droits du système et garantit la traçabilité.

2. Connexion et sécurité

Connexion. /sigep/login → e-mail + mot de passe + code MFA à 6 chiffres.
Compte le plus sensible du système. Protégez vos identifiants. Activez et imposez la double authentification (voir §11). Toute action est journalisée à votre nom.

3. Gestion des comptes

Menu Utilisateurs. Le Super Administrateur voit et gère tous les comptes.

3.1 Créer un compte

Étape 1. Utilisateurs → Créer un compte.
Étape 2. Choisir le niveau :
  • Administrateur délégué — accès configurable à la carte (voir §4).
  • Stratégique — statistiques agrégées, sans données individuelles.
  • Juge — gestion judiciaire des dossiers de sa juridiction.
  • Opérationnel — agent de terrain, dossiers assignés.
Étape 3. Renseigner identité, e-mail, mot de passe (≥ 8 caractères), badge, juridiction. Pour un administrateur délégué : cocher les accès. Créer.
Ce que cela implique : un compte auth est créé ; l'utilisateur peut se connecter ; l'action est journalisée (CREATE_USER). Un juge créé devient autonome sur ses dossiers ; un agent doit être assigné à des dossiers par un juge.

4.2 Modifier les accès après création

Utilisateurs → ligne de l'administrateur → bouton « Permissions » → cocher/décocher → Enregistrer.
Ce que cela implique : les accès prennent effet immédiatement (menu et fonctions s'ouvrent/se ferment). Vous pouvez réduire le pouvoir à tout moment. Tracé : UPDATE_PERMISSIONS.
Un administrateur délégué ne voit que ce qui est coché. Le Super Administrateur, lui, a tout.

5. Suspension, réactivation, suppression

ActionEffet
DésactiverLe compte ne peut plus se connecter (session bloquée). Réversible.
RéactiverRétablit l'accès.
SupprimerSuppression définitive du compte et de son authentification.
Réinitialiser mot de passeEnvoie un lien de réinitialisation.
Irréversible. La suppression est définitive. Préférez la désactivation en cas de doute. Toutes ces actions sont journalisées.

6. Parc de bracelets (magasin)

Menu Bracelets. Inventaire complet : en ligne, hors ligne, batterie, SIM, dossier assigné.

6.1 Enregistrer un bracelet

Enregistrer un bracelet → saisir IMEI, N° SIM, modèle → Ajouter. Le bracelet entre en stock (non assigné).

6.2 Numéro SIM

Colonne N° SIM → cliquer → saisir le numéro de la carte SIM utilisée par le bracelet → ✓.
Burkina Faso. Notez l'opérateur (Orange BF / Moov Africa). L'APN doit correspondre (Orange : 613-02, Moov : 613-03). Sans APN correct, le bracelet n'a pas de données.

7. Assigner un bracelet à un dossier

Deux voies :

Depuis Bracelets. Ligne d'un bracelet disponible → Assigner → rechercher la personne/dossier → confirmer.
Depuis le dossier. Ouvrir le dossier → bloc Bracelet GPS → choisir un bracelet disponible → Assigner. Bouton Retirer pour le libérer.
Ce que cela implique : le bracelet est lié au dossier ; ses positions alimentent le suivi ; un bracelet déjà assigné ailleurs est libéré. Tracé : ASSIGN_DEVICE.

8. Balises BLE (beacons)

Bracelets → section Balises BLE. La balise domicile permet de confirmer la présence en intérieur (là où le GPS est faible).

Enregistrer un beacon. Saisir l'UID / MAC + libellé → Enregistrer.
Lier à un bracelet. Menu déroulant → choisir le bracelet. Réassigner = choisir un autre bracelet (libère l'ancien). Délier = icône chaîne.
Défaillant / swap. Icône ⚠️ = marquer défaillant ; lier un beacon de rechange au même bracelet.

8.1 Options d'alarme (icône ⚙️)

OptionEffet
Alarme activéeActive/désactive l'alarme de domicile.
Distance maxSeuil (m) au-delà duquel l'éloignement déclenche une alerte.
Délai de grâceDurée d'absence tolérée avant alerte (anti-faux-positif).
Horaires actifsPlage où l'alarme s'applique.
Domicile = position actuelleCapture la position du bracelet comme domicile de référence.
Ce que cela implique : l'alarme combine GPS (distance) et BLE (présence beacon). Si le beacon est détecté, l'éloignement GPS n'alarme pas (évite les faux positifs en intérieur). Les défauts s'appliquent aux futurs beacons.
Pré-requis BLE. Le bracelet doit avoir le Bluetooth activé (commande BLE). Vérifiez la détection (le beacon apparaît dans les données de scan) ; sinon, vérifiez pile/portée et activation.

9. Valider les périmètres définis par les juges

Quand un juge définit une obligation de périmètre, elle arrive en statut « À valider ».

Étape 1. Ouvrir le dossier (ou la page Géofences).
Étape 2. Tracer précisément la zone si nécessaire (outil de tracé) puis cliquer Valider.
Ce que cela implique : la géofence passe Active et est surveillée (alertes de sortie). Tracé : VALIDATE_GEOFENCE. Tant qu'elle n'est pas validée, elle n'est pas appliquée.

12. Santé de la flotte & supervision

Menu Bracelets — tuiles de synthèse :

  • En ligne / Hors ligne — connectivité actuelle.
  • Batterie faible — bracelets sous 20 %.
  • Sans contact > 24 h — bracelets muets (à investiguer : SIM, batterie, hors zone réseau).
  • Non assignés — stock disponible.
Burkina Faso. Un « sans contact » peut venir d'une zone à faible couverture ou d'un délestage. Vérifiez l'historique avant de conclure à une fraude.

13. Journal d'audit global

Menu Journal d'audit — réservé au Super Administrateur (et admin habilité). Chaque entrée : qui (nom + rôle), action, cible, quand.

Actions tracées : création/suppression/suspension de comptes, modification de permissions, création de dossier, statut, géofences (définir/valider/supprimer), assignation bracelet, beacons, commandes, mesure (prolonger/mainlevée), paramètres.

Ce que cela implique : traçabilité totale, y compris des actions du Super Administrateur lui-même. Le journal est en lecture seule (immuable).

14. Rétention & purge des données

Les durées de conservation sont définies dans les Paramètres. Une tâche de purge applique la politique.

Purge. Endpoint /api/cron/purge?secret=<CRON_SECRET>, à planifier quotidiennement (cron-job.org ou cron Vercel). Supprime les positions et entrées d'audit au-delà de la rétention.
Données pénales sensibles. Définissez la rétention conformément au cadre légal. La purge est définitive.

15. Sauvegarde & migration vers une infrastructure locale

En vue d'une souveraineté nationale des données (hébergement local au Burkina Faso) :

Export complet. node --use-system-ca scripts/export-all.mjs (avec les variables PG) → instantané JSON de toutes les tables dans backups/<horodatage>/.
Procédure complète. Voir docs/MIGRATION_LOCAL.md : création de la base cible via supabase/setup.sql, réimport, comptes auth, repointage des variables d'environnement, option serveur tracker local.
L'application est portable (pilotée par variables d'environnement) ; la base est du PostgreSQL standard. La migration est donc réalisable sans réécriture.

17. Bonnes pratiques de gouvernance

  • Moindre privilège. Créez des administrateurs délégués avec uniquement les accès nécessaires ; révisez régulièrement.
  • Séparation des pouvoirs. N'outrepassez pas les décisions judiciaires ; exécutez les obligations définies par les juges.
  • Sécurité. Imposez la 2FA, des mots de passe robustes, l'expiration de session.
  • Sauvegardes. Exportez régulièrement (avant toute opération sensible).
  • Audit. Consultez périodiquement le journal pour détecter toute anomalie.
  • Matériel. Tenez l'inventaire à jour ; suivez batteries et bracelets muets.
